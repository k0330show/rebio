# Rebio 実用化ロードマップ

Rebio を localStorage ベースのデモ MVP から実用版へ移行するための計画です。

## 現在の状態（v11時点）

- データはすべて単一ブラウザの `localStorage` に保存される
- ユーザー認証は無い（`CURRENT_USER` として田中花子を固定）
- 組織マスタ（ユーザー・部署・役職・シフト）はコード内に固定されたデモデータ
- ファイルは実アップロード不可（Libraryはモックプレビューのみ）

## フェーズ構成

| フェーズ | 内容 | 状態 |
|---|---|---|
| v12 | Supabase設計（本ドキュメント含む） | 設計のみ |
| v13 | Supabaseクライアント導入、データ永続化のSupabase対応 | 実装予定 |
| v14 | Supabase Storageによる実ファイルアップロード | 実装予定 |
| v15 | エラー処理・履歴機能・権限の見せ方・デプロイ手順 | 実装予定 |

## v13で実装すること

1. `app/lib/repositories/*` にデータアクセスの抽象層を新設する
2. 環境変数 `NEXT_PUBLIC_REBIO_DATA_MODE` で `local` / `supabase` を切り替える
3. Supabase未設定時は自動的に `local` モードにフォールバックする
4. 簡易的なログイン画面（`/login`）を追加する。Supabase未設定時はログイン不要でデモユーザーとして動作する

## v14で実装すること

1. Supabase Storageの `documents` バケットへの実ファイルアップロード
2. PDF・画像の簡易プレビュー（Officeファイルの全文プレビューは対象外のまま）
3. アップロード・削除時のエラーハンドリング

## v15で実装すること

1. 各種エラー状態（未設定・通信失敗・権限不足等）の丁寧な表示
2. ローディング・空状態の表示整備
3. 危険操作（削除）への確認ダイアログ
4. `activity_logs` を使った軽量な履歴表示
5. 権限（管理者/リーダー/一般メンバー/閲覧のみ）の考え方をドキュメント化
6. Vercel + Supabaseでのデプロイ手順

## 実用版で追加が必要になる可能性がある機能（このロードマップの範囲外）

- 通知（メール・プッシュ）
- 複数チーム間でのデータ連携
- 高度な権限管理（部署ごとの閲覧制限の細分化など）
- モバイルアプリ（PWA化含む）
- Officeファイルの編集・全文検索

これらは「Rebioをチャットアプリ化・重い管理システム化しない」という設計思想と相談しながら、
必要になった時点で個別に検討する。
